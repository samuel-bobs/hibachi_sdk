# Hibachi Python SDK v0.3.0 Release Notes

## Overview

Version 0.3.0 introduces **tick size enforcement** for order prices. The exchange now rejects orders where the price is not a multiple of the contract's tick size. The SDK validates prices client-side before submission and provides helper functions to round prices to valid increments.

This is a **breaking change** for users who submit prices at arbitrary precision. See the Migration Guide below.

---

## What's New

### Tick Size Enforcement

All contracts have a defined **tick size** (minimum price increment). Orders submitted with prices that are not exact multiples of the tick size will be rejected.

- **Client-side validation**: `place_limit_order`, `place_market_order`, `update_order`, and `batch_orders` now validate prices against the contract's tick size before sending the request. Invalid prices raise `ValidationError`.
- **Server-side enforcement**: The exchange API returns HTTP 400 for non-tick-aligned prices. This applies to limit prices, trigger prices, and order modifications.

### New Helper Functions

Three new utility functions are available for working with tick sizes:

```python
from hibachi_xyz import round_price_to_tick, round_quantity_to_step, check_tick_size

# Round a price to the nearest valid tick increment
tick_size = client.get_tick_size("BTC/USDT-P")
price = round_price_to_tick(mark_price * 0.99, tick_size)

# Round a quantity down to the nearest valid step increment
step_size = client.get_step_size("BTC/USDT-P")
quantity = round_quantity_to_step(desired_qty, step_size)

# Validate a price without rounding (raises ValidationError if invalid)
check_tick_size(price, tick_size)
```

### New Client Methods

- `client.get_tick_size(symbol)` - Returns the tick size as a `Decimal` for a given symbol
- `client.get_step_size(symbol)` - Returns the step size as a `Decimal` for a given symbol

## Migration Guide

### From v0.2.x to v0.3.0

Orders that previously succeeded with arbitrary-precision prices may now be rejected. Update your code to round prices before submission:

```python
# Before (may now raise ValidationError)
client.place_limit_order(
    symbol="BTC/USDT-P",
    quantity=0.001,
    price=float(mark_price),  # e.g., 71286.1234567
    side=Side.BID,
    max_fees_percent=max_fees,
)

# After
from hibachi_xyz import round_price_to_tick

tick_size = client.get_tick_size("BTC/USDT-P")
price = round_price_to_tick(float(mark_price), tick_size)

client.place_limit_order(
    symbol="BTC/USDT-P",
    quantity=0.001,
    price=price,
    side=Side.BID,
    max_fees_percent=max_fees,
)
```

The same applies to trigger prices and order modifications.

### Tick Sizes by Contract

Tick sizes are returned in the `/market/exchange-info` endpoint and can be retrieved via `client.get_tick_size(symbol)`. Example values:

| Symbol | Tick Size |
|--------|-----------|
| BTC/USDT-P | 0.1 |
| ETH/USDT-P | 0.01 |
| SOL/USDT-P | 0.001 |
